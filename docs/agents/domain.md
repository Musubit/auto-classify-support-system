# Domain Docs

工程技能在探索代码库时应如何消费领域文档。

## 探索前先读这些

- **`CONTEXT.md`** 在仓库根目录
- **`docs/adr/`** — 读取与当前工作领域相关的架构决策记录
- 如果上述文件不存在，**安静地跳过**，不要标记为缺失

## 文件结构

单上下文仓库：

```
/
├── CONTEXT.md
├── docs/adr/
│   ├── 0001-event-sourced-orders.md
│   └── 0002-postgres-for-write-model.md
└── src/
```

## 使用术语表中的词汇

当输出中提到领域概念时（issue 标题、重构提案、假设、测试名称），使用 `CONTEXT.md` 中定义的术语。不要漂移到术语表明确避免的同义词。

如果需要的概念不在术语表中，这是一个信号——要么你在发明项目不使用的语言（重新考虑），要么存在真正的空白（记录下来供后续补充）。

## 标记 ADR 冲突

如果输出与现有 ADR 矛盾，显式指出而不是默默覆盖：

> _与 ADR-0007 (event-sourced orders) 冲突 — 但值得重新讨论，因为…_
